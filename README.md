# Dns-rules

Готовые правила и конфиги для **sing-box**, позволяющие резолвить DNS-запросы отдельных приложений или доменов через [GeoHide](https://dns.geohide.ru:8443/) — DNS-сервер для обхода геоблокировок, — при этом весь остальной трафик системы идёт через обычный зашифрованный DNS (например, `dns.google`).

Репозиторий решает практическую задачу: не проксировать/туннелировать весь трафик устройства, а точечно подменять DNS только там, где это нужно (заблокированные по IP/DNS сервисы, гео-залоченные приложения), не трогая всё остальное.

## Содержимое репозитория

### `geohide.srs`
Скомпилированный бинарный rule-set (формат sing-box `.srs`) со списком доменов. Собирается автоматически через GitHub Actions (см. `.github/workflows` и `scripts`) на основе списка доменов из репозитория [Internet-Helper/GeoHideDNS](https://github.com/Internet-Helper/GeoHideDNS).

Подключается в конфиге sing-box как remote rule-set:

```json
{
  "type": "remote",
  "tag": "geohide-domains",
  "format": "binary",
  "url": "https://github.com/Kira361323/Dns-rules/releases/latest/download/geohide.srs",
  "update_interval": "24h"
}
```

### Готовые конфиги sing-box (`*.json`)

Все конфиги используют одну и ту же схему: `dns.google` (DoT) — сервер по умолчанию для всей системы, `dns.geohide.ru` (DoH) — сервер для выбранных приложений и/или доменов. Различаются только тем, что именно и как направляется на GeoHide.

| Файл | Что направляется на GeoHide | Как |
|---|---|---|
| `1-only-apps.json` | Только приложения из списка (по `package_name`) | Одно DNS-правило с массивом пакетов Android-приложений |
| `2-only-domains.json` | Только домены из списка | DNS-правило, ссылающееся на `geohide.srs` как remote rule-set |
| `3-combined-logical.json` | Приложения **и** домены из списка | Одно DNS-правило типа `logical` с режимом `or`, объединяющее оба условия |
| `4-combined-separate-rules.json` | Приложения **и** домены из списка | Два отдельных DNS-правила подряд, оба ведут на GeoHide (функционально эквивалентно варианту 3, но проще читать) |

Список приложений в правилах: ChatGPT, Claude, Copilot, DeepL, DeepSeek, DuckDuckGo, Firefox, Gemini, Google App, Grok, Leonardo.Ai, Mistral, Google Tailwind, Qwen, Suno, Telegram Beta.

### `singbox-dns-only.json`
Исходный базовый конфиг, с которого начался репозиторий. Тоже вариант «только для приложений из списка» (как `1-only-apps.json`), без remote rule-set для доменов — список приложений задан вручную прямо в DNS-правиле. Можно использовать как минимальный референс-конфиг или отправную точку для собственных правок.

## Как использовать

1. Скачайте нужный `.json` из этого репозитория.
2. Подставьте его как основной конфиг sing-box (или возьмите из него блоки `dns` / `route` для своего конфига).
3. При необходимости отредактируйте список `package_name` под свои приложения — актуальные имена пакетов можно посмотреть на странице приложения в Google Play (в URL после `id=`).
4. `geohide.srs` подтягивается автоматически по ссылке из релиза — обновлять вручную не нужно, но при желании можно уменьшить/увеличить `update_interval`.

## Дисклеймер

Конфиги подменяют только DNS-резолвинг для указанных доменов/приложений. Это не туннелирование и не проксирование трафика — если сервис блокируется не по DNS, а по IP или SNI, эти конфиги не помогут.
